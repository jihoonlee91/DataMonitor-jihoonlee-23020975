# 작업 보고서 - DataMonitor PoC

## 요청 받은 작업

"데이터 모니터링 Tool" PoC. 현재 저장된 데이터 상태를 콘솔에서 실시간 조회할 수 있는 관리자 도구 구현.

## 해석

- 최종 프로젝트의 모니터링 메뉴 명세(상태별 주문 수, 시료별 재고 현황, REJECTED 제외)를 미리 반영하여
  PoC 단계에서부터 실제 요구사항과 동일한 표시 항목을 검증하고자 함.
- 다른 PoC가 생성한 데이터를 그대로 읽을 수 있어야 실전에서 여러 도구가 같은 DB를 공유하는 상황을 검증할 수 있다고 판단하여
  `--data-dir` 옵션을 필수로 설계함.

## 변경 요약

- `monitor.py`: JSON 로드, 상태별 주문 집계, 재고 상태(여유/부족/고갈) 판정, 1회/반복 조회 모드
- `data/`: 데모용 샘플 데이터(실제 운영 데이터 아님)

## 핵심 코드 요약

- `render()` 함수가 조회 결과를 하나의 문자열로 조립하여 1회 조회와 반복 조회(`clear_screen` 후 재출력) 양쪽에서 재사용된다.
- REJECTED 상태는 집계는 하되 "모니터링 제외" 주석을 붙여 요구사항(REJECTED는 유효 주문 아님)을 코드 주석이 아닌 출력 문구로 드러낸다.
